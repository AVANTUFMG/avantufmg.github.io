Introdução

Essa documentação tem como objetivo demonstrar os processos básicos necessários para o desenvolvimento dos softwares envolvidos na construção do MAV da Avant UFMG. Os procedimentos serão organizados em dois sistemas: o sistema embarcado (PX4 Firmware) e o sistema de controle em solo (QGroundControl/Mission Planner).

Plataformas Suportadas

Geral

Todas as etapas demonstradas a seguir foram executadas no sistema operacional Microsoft Windows 7 Home Basic x64 Service Pack 1, e portanto todos os links se destinam a essa plataforma. Apesar disso, todo o conjunto de software pode ser utilizado em Linux ou Mac OSX.

O próximo passo é inicializar o código fonte utilizando o comando:

git submodule update --init --recursive

Nesse momento, o código fonte está pronto para ser editado e compilado para o firmware da PixHawk. Em seguida, baixaremos a IDE que será utilizada para facilitar esses procedimentos.

Eclipse IDE


Apesar de ser possível usar o make para compilar diretamente o firmware, o uso de um ambiente de desenvolvimento integrado (IDE) facilita muito o processo de escrever e editar o código. Usaremos o Eclipse, um IDE de código aberto que suporta desenvolvimento em inúmeras linguagens de programação como Java, C/C++, PHP, Python e Android.

Eclipse 64bit para Windows

Após o download, basta executar o arquivo e instalar o programa em seu destino padrão.

Utilizando waf no Windows com Cygwin


Waf é uma ferramenta de automação de compilação que, em poucas palavras, permite que um código seja prontamente compilável em diversos sistemas operacionais, com suas configurações e compiladores próprios, de forma automatizada. O código do ardupilot utiliza waf para sua compilação. Para utilizar waf no Windows, será necessária a instalação do Cygwin.

  Observação

Há um script pré-pronto na pasta /ardupilot/Tools/environment_install/install-prereqs-windows.ps1 que executa todos os passos descritos abaixo.

Instalando Cygwin

  1. Abra https://www.cygwin.com/install.html no seu navegador e baixe "setup-x86_64.exe".
  2. Abra o propmpt de comando (pressionando as teclas ⊞ Win + D e digitando "cmd", e então "OK") e navegue para o diretório onde você baixou o arquivo de download (use o comando cd para mudar de diretório e dir para ver o diretório em que está).
  3. Uma vez que tenha navegado até o diretório onde está o arquivo "setup-x86_64.exe" baixado, entre com o código abaixo: 
    setup-x86_64.exe -P autoconf,automake,ccache,gcc-g++,git,libtool,make,gawk,libexpat-devel,libxml2-devel,python36,python36-future,python36-lxml,python36-pip,libxslt-devel,python36-devel,procps-ng,zip,gdb,ddd

  Observação

Algumas vezes a instalação pode ficar congelada por conta de seu anti-vírus. Se isso acontecer, tente fechar todos os outros programas abertos, incluindo o anti-vírus, e tente novamente.


Instalando o compilador GCC

  • Baixe e instale o compilador gcc-arm-non-eabi em firmware.ardupilot.org/Tools/STM32-tools (link direto aqui)
    • aceite todas as licenças
    • instale no diretório padrão
    • aceite todos os certificados ssl
    • marque a opção “Add path to environment variable”
Instalando MAVProxy para Windows

O MAVProxy é necessário se for utilizar o SITL (Software In The Loop). Se você não for utilizar o SITL, pode pular essa etapa.

  1. Para instalar o MAVProxy para Windows, siga a documentação do MAVProxy.
Configurando diretórios/paths e pacotes extras no Cygwin
  1. abra e feche o programa “Cygwin64 Terminal” a partir da área de trabalho ou menu de Inicialização do Windows. Isso irá criar os arquivos de inicialização para o usuário no diretório principal do Cygwin.
  2. abra o “Cygwin64 Terminal” novamente. Entre com os seguintes comandos para setar Python 3.6 como a versão padrão de Python e então instalar os pacotes adicionais de Python:
    3. ln -s /usr/bin/python3.6 /usr/bin/python
ln -s /usr/bin/pip3.6 /usr/bin/pip
pip install empy pyserial pymavlink
Compile com Waf

Você deve ser capaz, agora, de iniciar o programa “Cygwin64 Terminal” a partir do menu de Inicialização do Windows e compilar com Waf, como descrito em BUILD.md.

  Observação

A compilação pode falhar se o caminho para alguns arquivos for muito longo. Se a compilação falhar, tente clonar o repositório do ardupilot numa posição alta na sua estrutura de diretórios (por exemplo: ~/ardupilot ou C:/ardupilot).

Se você tiver um de "missing packet algumacoisa", verifique se digitou corretamente o comando no cmd mencionado anteriormente, pois ele que garante que todos os pacotes foram instalados.

Configurando o Eclipse


Após a instalação, abra o Eclipse através do atalho criado no menu iniciar e escolha o workspace padrão.


Criaremos agora um novo projeto para organizar o código fonte:

  • Vá até File>New>Makefile Project with Existing Code.
  • Em "existing code location", escolha a pasta ArduPlane (nosso firmware usa a versão de asa fixa) , dentro da pasta de onde foi baixado o firmware ArduPilot do GitHub.
  • Em "ToolChain for Indexer Settings" escolha Cygwin GCC.

  • Em seguida, no Project Explorer na lateral direita, clique com o botão direito na pasta do projeto e escolha Properties.
    Quando a janela "Properties for ardupilot" aparecer, vá em “C/C++ Build”, desmarque “Use default build command” e ponha c:\cygwin64\bin\python2.7 waf no campo “Build command” como mostrado na figura abaixo:


  • Na aba “Build Targets” à direita selecione “ardupilot” e pressione o botão verde “New Build Target”.
  • No campo “Target name” entre com configure --board fmuv3 --no-submodule-update e pressione OK para criar um build target para configurar o waf para compilar para controladores de vôo Pixhawk/Cube usando ChibiOS
  • Crie outro build target chamado plane
  • Rode estes dois targets para criar um arquivo binário do ArduPilot Plane para controladores de vôo Pixhawk/Cube
  • Se a compilação falhar com um aviso “Missing configuration file” cheque o diretório do ardupilot directory por um diretório chamado “build”, delete-o e tente novamente
  • Outros build targets úteis são, clean, plane, rover and --help

O firmware arducopter.apj será criado na pasta ardupilot\build\fmuv3\bin e pode ser enviado a partir do Mission Planner ou de outro sistema de estação solo para o controlador.

Editando e compilando o firmware

Com o projeto aberto e configurado no Eclipse, é possível modificar qualquer arquivo através do Project Explorer.
Após um longo processo de compilação (Ou não, se vc for o Iure), o arquivo de firmware será criado na pasta ArduPlane com a extensão .apj e pode ser carregado para a pixhawk usando o MissionPlanner na opção de "custom firmware".

Mission Planner


Programa pré-compilado


Apesar da estação-solo utilizada em nossos testes ser o qGroundControl, o Mission Planner é importante por sua maior capacidade de customização, e principalmete a opção de carregar um firmware customizado. Em um primeiro momento, talvez seja interessante usar a versão pronta do Mission Planner. Ela pode ser baixada no link:

Mission Planner


Com o intuito de tornar esta documentação mais completa, os passos a seguir orientam a compilar o Mission Planner a partir da fonte. Com isso, a equipe pode personalizar a estação.

Repositório do GitHub

Use o comando

git clone https://github.com/ArduPilot/MissionPlanner

para clonar o repositório.

Compilando a partir do código fonte


Para compilar o Misson Planner, é necessário possuir o Microsoft Visual Studio 2017 instalado no computador. A IDE é gratuita e pode ser baixada no link:

Microsoft Visual Studio 2017

Escolha a versão "Visual Studio Community 2017". Após abrir o executável, o instalador irá mostrar a tela de seleção de cargas de trabalho:


Marque as opções "Desenvolvimento para desktop com o .NET" em Windows e "Desenvolvimento de multiplataforma do .NET Core" em Outros conjuntos de ferramentas e as instale. Certifique-se de possuir o espaço em disco necessário para os componentes.

Após o download, inicie o Visual Studio e vá em Arquivo>Abrir>Projeto/solução. Navegue até a pasta onde foi baixado o repositório no item acima e escolha "MissionPlanner.sln". No canto superior, escolha "Release" e "x86". (x64 causou erro no meu computador, ainda não sei porque).


O projeto agora está configurado e pode ser modificado de acordo com as necessidades do programador. Um exemplo básico é simplesmente personalizar um gráfico da interface. Nessa demonstração, substituiremos a logo da ArduPilot por uma logo customizada. Para realizar esta modificação, copie o arquivo de imagem desejado para a pasta MissionPlanner/Resources. Em seguida, no Visual Studio, localize o arquivo resources.resx dentro da pasta Properties:


Este arquivo contém as strings que apontam para os arquivos de imagem do programa. O arquivo a ser trocado está localizado na última linha. Basta substituir a linha: 

 <value>..\Resources\ 0d92fed790a3a70170e61a86db103f399a595c70.png;System.Drawing.Bitmap,
 System.Drawing, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a </value>

Por:

 <value>..\Resources\avant.png;System.Drawing.Bitmap,
 System.Drawing, Version=4.0.0.0, Culture=neutral, PublicKeyToken=b03f5f7f11d50a3a</value>

Onde avant.png é o nome do arquivo de imagem escolhido. O resultado ficará:
